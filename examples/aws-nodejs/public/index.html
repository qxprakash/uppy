AWS S3 upload example

Demonstrates the @uppy/aws-s3 plugin with two signing modes.

Regression Verification Mode
Upload 3+ files > 100MB. Rapidly pause/resume one file mid-upload. Then check:
#4313 Console summary after complete — each file must have a unique uploadId
#5667 Console — each part must show only [PartNumber, ETag]
#5328 Network tab — find POST ?uploadId=..., check XML has proper <Part> elements
#4648 Console — pause/resume logs with no TypeError
#3447 Console — "upload-success" must appear exactly once per file
Sign on the client (STS credentials)

Uses getCredentials to fetch temporary STS credentials from /s3/sts. The browser signs all S3 requests locally using SigV4.

Sign on the server (presigned URLs)

Uses signRequest to call /s3/presign for each S3 operation. The server generates presigned URLs; the browser uses them directly.
